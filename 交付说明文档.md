## 五、常见问题排查

### 5.1 用户无法检测到新版本

**可能原因：**

| 检查项 | 解决方法 |
|--------|----------|
| GitHub Secrets 未配置 | 确认 `TAURI_SIGNING_PRIVATE_KEY` 已配置 |
| latest.json 未更新 | 检查 GitHub Pages 部署状态 |
| 签名验证失败 | 检查 `.sig` 文件是否存在于 Release |
| 网络问题 | 检查是否能访问 GitHub |

**排查命令：**
```bash
# 检查 latest.json
curl https://MiMo-ts.github.io/kuaifan-claw/latest.json

# 检查签名文件
curl -I https://github.com/MiMo-ts/kuaifan-claw/releases/download/v1.0.23/claw_1.0.23_x64-setup.exe.sig
```

### 5.2 签名验证失败

**错误信息：** `Signature verification failed`

**解决方法：**
1. 确认 GitHub Secrets 中的私钥与 `tauri.conf.json` 中的公钥配对
2. 私钥泄露或更换后，需要用户重新安装旧版本

### 5.3 构建失败

**检查项：**
- [ ] GitHub Secrets 存在且有效
- [ ] `npm run tauri:build` 本地能成功
- [ ] Node.js / Rust 版本正确

### 5.4 邀请码验证失败

**可能原因：**
- 后端服务未运行
- 数据库表结构不完整
- API URL 配置错误

**排查命令：**
```bash
# 检查后端健康状态
curl http://kuaifandl.asia/health

# 检查数据库表
USE agent_backend;
SHOW TABLES;
DESCRIBE invitecodes;
```

## 六、GitHub Actions 工作流

### 6.1 Release 工作流 (.github/workflows/release.yml)

```
推送 tag v* → GitHub Actions 触发
       ↓
┌─────────────────────────────────┐
│ 1. Checkout 代码                │
│ 2. Setup Node.js / Rust        │
│ 3. 安装依赖 (npm ci)           │
│ 4. 构建前端 (npm run build)     │
│ 5. 构建 Tauri 应用              │
│ 6. 生成签名 (.sig)             │
│ 7. 生成校验和 (checksums.txt)  │
│ 8. 生成 update JSON             │
│ 9. 创建 Release                 │
│ 10. 部署 latest.json 到 Pages   │
└─────────────────────────────────┘
```

### 6.2 构建产物

| 产物 | 位置 | 用途 |
|------|------|------|
| NSIS 安装包 | `bundle/nsis/*.exe` | 分发用户 |
| 校验和 | `checksums.txt` | 验证完整性 |
| 签名文件 | `*.exe.sig` | 签名验证 |
| 更新 JSON | `update-*.json` | 版本检测 |
