## 八、邀请码验证迁移到首页（2026-04-23 新增）

### 需求
用户完成向导页三步骤后，首次进入首页需要验证邀请码。

### 修改内容

**1. 新增 Rust 命令**

`src-tauri/src/commands/invite.rs` - 新增 `is_invite_code_validated` 函数：

```rust
/// 检查邀请码是否已验证
#[command]
pub async fn is_invite_code_validated(data_dir: String) -> Result<bool, String> {
    let data_path = PathBuf::from(&data_dir);
    crate::services::invite_code::is_invite_code_validated(&data_path)
}
```

**2. src-tauri/src/main.rs - 新增命令注册**

```rust

```

**3. App.tsx 修改**

原代码：
```typescript
const init = async () => {
  try {
    const dataDir = await invoke<string>('get_data_dir');
    const storedStatus = localStorage.getItem('inviteCodeValidated');
    if (storedStatus === 'true') {
      setInviteCodeValidated(true);
    }
  } catch (e) {
    console.error('Init error:', e);
  } finally {
    setCheckingInviteCode(false);
    setInitialized(true);
  }
};
```

修改后：
```typescript
const init = async () => {
  try {
    const dataDir = await invoke<string>('get_data_dir');
    // 通过 Rust 命令检查邀请码是否已验证
    const isValid = await invoke<boolean>('is_invite_code_validated', { dataDir });
    setInviteCodeValidated(isValid);
    if (isValid) {
      localStorage.setItem('inviteCodeValidated', 'true');
    }
  } catch (e) {
    console.error('Init error:', e);
    const storedStatus = localStorage.getItem('inviteCodeValidated');
    if (storedStatus === 'true') {
      setInviteCodeValidated(true);
    }
  } finally {
    setCheckingInviteCode(false);
    setInitialized(true);
  }
};
```

修改条件判断：
```typescript
// 之前
if (!inviteCodeValidated) {
  return <InviteCodePage />;
}

// 之后 - 只有向导完成后才检查邀请码
if (wizardCompleted && !inviteCodeValidated) {
  return <InviteCodePage />;
}
```

## 九、Workflow Push 失败修复（2026-04-23 新增）

### 问题
推送 v1.0.39 时，workflow 报错：
```
[rejected] HEAD -> main (fetch first)
error: failed to push some refs
```

原因是 workflow 在 detached HEAD 状态推送时，远程 main 有新的提交。

### 修复内容

`.github/workflows/release.yml` - `Commit update-latest.json to main branch` 步骤：

原代码：
```yaml
git config --local user.email "github-actions[bot]@users.noreply.github.com"
git config --local user.name "github-actions[bot]"
git add "./update-latest.json"
git commit -m "chore: update latest.json for ${{ github.ref_name }}"
git push origin HEAD:main
```

修改后：
```yaml
git config --local user.email "github-actions[bot]@users.noreply.github.com"
git config --local user.name "github-actions[bot]"
git add "./update-latest.json"
git commit -m "chore: update latest.json for ${{ github.ref_name }}"
# Fetch latest main and rebase before pushing
git fetch origin main
git rebase origin/main
# Push current commit to main branch (use HEAD:main since we're in detached HEAD state)
git push origin HEAD:main
```
