## 二、技术实现

### 2.1 核心依赖

#### Rust 后端 (src-tauri/Cargo.toml)
```toml
tauri-plugin-updater = "2"   # 自动更新插件
tauri-plugin-process = "2"  # 进程管理（用于重启应用）
```

#### 前端 (web/package.json)
```json
"@tauri-apps/plugin-updater": "latest",
"@tauri-apps/plugin-process": "latest"
```

### 2.2 配置文件

#### tauri.conf.json 更新配置
```json
{
  "plugins": {
    "updater": {
      "pubkey": "YOUR_PUBLIC_KEY_HERE",
      "endpoints": [
        "https://github.com/MiMo-ts/kuaifan-claw/releases/download/{{target}}/{{arch}}/{{current_version}}.json"
      ],
      "windows": {
        "installMode": "passive"
      }
    }
  }
}
```

**endpoint 变量说明：**
- `{{target}}` - 目标平台（如 windows）
- `{{arch}}` - 架构（如 x86_64）
- `{{current_version}}` - 当前版本号

### 2.3 更新 JSON 格式

每个版本发布时，GitHub Actions 会自动生成如下 JSON：

```json
{
  "version": "1.0.2",
  "notes": "快泛claw v1.0.2 发布\n\n新增功能：\n- xxx\n- xxx\n\n修复：\n- xxx",
  "platforms": {
    "windows-x86_64": {
      "signature": "...",
      "url": "https://github.com/USER/REPO/releases/download/v1.0.2/快泛claw-1.0.2-x64-setup.exe"
    }
  }
}
```

### 2.4 前端更新流程

```typescript
// web/src/utils/updater.ts

import { check } from '@tauri-apps/plugin-updater';
import { relaunch } from '@tauri-apps/plugin-process';

// 1. 检查更新
const update = await check();
if (update?.available) {
  // 2. 下载并安装
  await update.downloadAndInstall((event) => {
    if (event.event === 'Progress') {
      const percent = Math.round(
        (event.data.chunkLength / event.data.contentLength) * 100
      );
      // 更新进度 UI
    }
  });

  // 3. 重启应用
  await relaunch();
}
```

### 2.5 GitHub Actions 自动构建流程

```
push tag → GitHub Actions 触发 → 自动化构建 → 发布 Release
```

**工作流程 (.github/workflows/release.yml)：**
1. 检出代码
2. 安装 Node.js、Rust 工具链
3. 构建前端 (`npm run build`)
4. 构建 Tauri 应用 (`tauri build`)
5. 生成 SHA256 校验和
6. 生成 update JSON
7. 创建 GitHub Release 并上传产物

## 三、安全机制

### 3.1 签名验证

更新包采用 **Ed25519 签名**机制：

1. **私钥** - 存储在 GitHub Secrets，仅 GitHub Actions 可访问
2. **公钥** - 内置于应用代码中

每次更新时：
- GitHub Actions 用私钥对安装包签名
- 客户端用公钥验签，确保包来源可信

### 3.2 校验和验证

除签名外，还提供 SHA256 校验和验证，防止下载过程中被篡改。

## 四、使用说明

### 4.1 开发人员 - 发布新版本

#### 前提条件

1. GitHub 仓库已创建
2. 已配置 GitHub Secrets（详见「配置」章节）
3. 代码已完成开发和测试

#### 发布步骤

```bash
# 1. 确保所有更改已提交
git add .
git commit -m "feat: xxx"

# 2. 拉取最新代码
git pull origin main

# 3. 本地构建测试（可选，确保无编译错误）
npm run tauri:build

# 4. 打版本标签（格式：v*.*.*）
git tag v1.0.2

# 5. 推送标签到 GitHub
git push origin v1.0.2
```

#### GitHub Actions 自动执行

标签推送后，GitHub 会自动：
```
✓ 构建前端
✓ 构建 Tauri 应用
✓ 生成 checksums.txt
✓ 生成 update JSON
✓ 创建 GitHub Release
✓ 上传所有产物
```

可在 GitHub 仓库 → Actions 页面查看构建进度。

### 4.2 用户 - 安装/更新

#### 首次安装

1. 从 GitHub Releases 页面下载最新版本的安装包
2. 双击运行安装程序
3. 按照提示完成安装

#### 后续更新

**方式一：自动更新（推荐）**

1. 打开已安装的应用
2. 如果有新版本，顶部会出现蓝色更新横幅
3. 点击「更新」按钮
4. 等待下载完成，应用自动重启
5. 更新完成

**方式二：手动更新**

1. 前往 GitHub Releases 页面
2. 下载最新版本的安装包
3. 直接运行新版本安装包，覆盖安装

## 五、配置指南

### 5.1 生成签名密钥

在项目根目录执行：

```bash
cd D:\ORD
npm run tauri signer generate -- --rsa 4096
```

按照提示输入密码，会生成两个文件。

### 5.2 配置 GitHub Secrets

1. 打开 GitHub 仓库页面
2. 进入 Settings → Secrets and variables → Actions
3. 点击 New repository secret，添加：

| Secret Name | Value 来源 |
|-------------|-----------|
| `TAURI_SIGNING_PRIVATE_KEY` | `~/.tauri/YOUR_APP_NAME.key` 文件内容 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 密钥生成时设置的密码 |

### 5.3 配置 tauri.conf.json

编辑 `src-tauri/tauri.conf.json`，将 `PLACEHOLDER_REPLACE_WITH_YOUR_PUBLIC_KEY` 替换为公钥内容：

```json
"plugins": {
  "updater": {
    "pubkey": "dW50cnVzdGVkIGNvbW1lbnQ6IG1pbmlzaWduIHB1YmxpYyBrZXk6IE...",
    "endpoints": [
      "https://github.com/MiMo-ts/kuaifan-claw/releases/download/{{target}}/{{arch}}/{{current_version}}.json"
    ]
  }
}
```

同时修改 `identifier` 为你的 GitHub 仓库路径：
```json
"identifier": "com.yourname.yourrepo"
```

### 5.4 修改 GitHub Actions 构建配置

编辑 `.github/workflows/release.yml`，确认仓库名称正确：

```yaml
- name: Create update JSON
  shell: pwsh
  run: |
    # 确认这里的仓库名称是你的
    $json.platforms."windows-x86_64".url = "https://github.com/MiMo-ts/kuaifan-claw/releases/download/${{ github.ref_name }}/$($exe.Name)"
```

## 七、Troubleshooting

### 构建失败

**问题**：GitHub Actions 构建失败

**解决**：
1. 查看 Actions 日志确定失败原因
2. 常见问题：
   - 密钥格式错误 - 确保 Secrets 中的私钥是单行字符串
   - 依赖下载失败 - 重试通常可以解决
   - 签名失败 - 检查私钥密码是否正确

### 更新检测不到

**问题**：发布新版本后，客户端检测不到更新

**解决**：
1. 确认 GitHub Release 已创建成功
2. 确认 update JSON 文件已上传
3. 检查 endpoint URL 是否正确
4. 检查版本号是否递增（必须大于当前版本）

### 签名验证失败

**问题**：`Signature verification failed`

**解决**：
1. 确认公钥与私钥配对
2. 确认私钥未泄露或更换
3. 如果更换密钥，需要用户重新安装旧版本后再更新
